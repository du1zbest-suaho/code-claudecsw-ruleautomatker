# 보험 사업방법서 속성 추출 자동화 - 작업일지

> **목적**: 한화생명 보험 상품 사업방법서(PDF)에서 가입조건 정보를 자동 추출하여 시스템 업로드 양식(xlsx)으로 변환하는 AI 에이전트 개발 과정 기록

---

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 목표 | PDF 사업방법서 → 시스템 업로드 양식 자동화 |
| 처리 대상 | 51개 PDF (한화생명 판매 중 상품) |
| 추출 테이블 | S00026(가입가능나이), S00027(보험기간/납입기간), S00028(납입주기), S00022(보기개시나이) |
| 검증 기준 | 기존 DB (판매중_* 파일) Ground Truth |
| 도구 | Python + Claude AI (LLM 기반 추출) |

---

## 핵심 아키텍처

```
ZIP(사업방법서)
    ↓ extract_pdf.py (텍스트+이미지 추출)
    ↓ scan_keywords.py (관련 페이지 식별)
    ↓ extraction_rules.py (LLM/룰 기반 정보 추출)
    ↓ convert_codes.py (자연어→시스템코드 변환)
    ↓ validate (GT와 비교 검증)
    ↓ generate_upload.py (xlsx 업로드 양식 생성)
```

**목표**: S00022 미추출 9건 → 0건

**배경**:
S00022는 보험개시 시점 정보. 크게 2가지 유형:
- **X-type**: "연금개시나이 N세~M세" → X55, X56, ... X80 개별 행
- **N-type**: "계약일 이후 N년 경과시점" → N2, N3, N5 등 (종신보험 스마트연금전환특약)

**해결책**:
```python
# N-type 패턴 추가
for m in re.finditer(r"계약일\s*(?:부터|이후)\s*(\d+)년\s*경과시점", text):
    n_years.add(int(m.group(1)))
for m in re.finditer(r"전환일\s*(?:부터|이후)\s*(\d+)년\s*경과시점", text):
    n_years.add(int(m.group(1)))
```

**추가 개선**:
- 다중 종별 연금개시나이 테이블 파싱 (1종: X55~X80, 2종: X55~X110)
- S00027 전기납 → X{연금개시나이} 확장 (연금보험 PP 코드화)

## 주요 기술적 도전과 해결 사례

### 1. PDF 텍스트 추출 아티팩트
**문제**: PDF 변환 시 "20\n세" → "20세" 인식 불가
**해결**: 정규식 전처리 `digit\n세` → `digit세`

### 2. 희소(Sparse) 테이블
**문제**: 일부 셀이 `-`인 테이블에서 행 구조 파악 불가
**해결**: `_parse_age_table_sparse_multiline` 특수 파서 개발

### 3. 성별 중립 행 자동 생성
**문제**: GT에는 "남녀공통" 행이 있지만, PDF에는 남/여 별도 행만 존재
**해결**: 남/여 동일 조건 → 남녀공통 행 자동 생성 (`_add_gender_neutral_rows`)

### 4. 갱신형 vs 종신형 구분
**문제**: "재가입 종료 나이" 근처의 "종신" 텍스트를 종신만기로 오인
**해결**: 200자 근접도 기반 컨텍스트 필터

### 5. N-type 보기개시나이
**문제**: 종신보험 스마트연금전환특약의 보기개시 시점이 "계약일 이후 N년"으로 기재
**해결**: N-type 패턴 신규 추가, SPIN_STRT_DVSN_CODE='N' 코드 변환

### 6. 연금보험 전기납 코드화
**문제**: 연금보험의 납입기간 "전기납"은 연금개시나이별로 X{age} 코드로 분리 필요
**해결**: `_extract_annuity_onset_range`로 연금개시나이 범위 파악 후 전기납 확장
